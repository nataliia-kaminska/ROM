# Research Opportunity Matcher

## Академічний опис проєкту

**Research Opportunity Matcher** — це веборієнтована інформаційна система для пошуку, нормалізації, персоналізованого ранжування та супроводу заявок на наукові можливості: гранти, стипендії, програми академічної мобільності, дослідницькі позиції, навчальні програми та конкурси для дослідників.

Метою системи є зменшення часу, який дослідник витрачає на ручний пошук релевантних програм, і підвищення якості прийняття рішень щодо подання заявки. Система поєднує імпорт даних із відкритих джерел, профіль дослідника, повнотекстовий пошук, семантичне порівняння, дошку статусів, нагадування та AI-асистента підготовки заявки.

## Основні функціональні можливості

Система реалізує такі ключові можливості:

- автентифікація користувача через локальний акаунт або ORCID OAuth 2.0;
- створення та редагування профілю дослідника;
- імпорт відкритих академічних даних із ORCID та OpenAlex;
- пошук публічних профілів дослідника через зовнішній вебпошук із підтвердженням користувача;
- імпорт можливостей із Grants.gov, EU Funding & Tenders, Horizon Europe, MSCA, Erasmus+, NAWA та інших джерел;
- нормалізація назв, описів, дедлайнів, країн, ключових слів і вимог;
- дедуплікація можливостей за URL, зовнішнім ідентифікатором та ключовими метаданими;
- персоналізоване ранжування можливостей за match score;
- повнотекстовий пошук через Elasticsearch або fallback-пошук у базі даних;
- семантичне порівняння профілю та можливостей через embeddings;
- Kanban-дошка для статусів заявки;
- система нагадувань за дедлайнами;
- центр сповіщень;
- AI Apply Assistant для формування memo, чеклистів, ризиків і підготовчих дій;
- адміністративна панель імпорту, моніторингу та аналітики.

## Архітектура системи

Проєкт побудований як клієнт-серверний застосунок із розділенням відповідальностей між frontend, backend, базою даних, фоновими задачами та зовнішніми сервісами.

Backend має модульну структуру, наближену до Clean Architecture:

- **API layer** — FastAPI routers, залежності, авторизація, HTTP-контракти;
- **Application / Services layer** — use cases, бізнес-логіка, рекомендації, імпорт, профільне збагачення;
- **Domain layer** — доменні сутності, enum-типи та правила предметної області;
- **Infrastructure layer** — SQLAlchemy-моделі, репозиторії, інтеграції з ORCID, OpenAlex, Grants.gov, Elasticsearch, Redis, SMTP;
- **Workers** — фонові задачі для імпорту, нагадувань, дайджестів та оновлення embeddings.

Frontend побудований на React і розділений на:

- `views` — сторінки застосунку;
- `components` — повторно використовувані UI-компоненти;
- `hooks` — стан сесії, профілю, робочого простору, сповіщень і асистента;
- `api` — клієнтські обгортки над backend API;
- `styles` — базові стилі, форми, компоненти, сторінки та адаптивність.

## Технологічний стек

У проєкті використано:

- **Python 3.11+**;
- **FastAPI** для backend API;
- **SQLAlchemy 2.x** для роботи з базою даних;
- **Alembic** для міграцій;
- **PostgreSQL** як основна база даних;
- **pgvector** для зберігання та порівняння embeddings;
- **Redis** для черг і realtime-комунікацій;
- **RQ / worker-процеси** для фонових задач;
- **Elasticsearch** для повнотекстового пошуку;
- **React + TypeScript + Vite** для frontend;
- **SMTP / Mailtrap** для email-перевірки й тестової доставки повідомлень;
- **Groq API або локальні/детерміновані fallback-механізми** для AI-збагачення;
- **Docker Compose** для контейнеризованого запуску повного стеку.

## Роль профілю дослідника

Профіль дослідника є центральним елементом персоналізації. Він містить:

- повне ім’я;
- email;
- career stage;
- країну базування або первинної eligibility;
- дисципліни;
- ключові слова;
- бажані країни;
- науковий опис;
- публікації;
- ступені;
- мови;
- funding interests;
- небажані або недоступні країни;
- бажані типи можливостей.

Ці дані використовуються для:

- обчислення match score;
- пояснення відповідності;
- оцінки готовності заявки;
- генерації AI memo;
- формування рекомендацій;
- побудови embeddings профілю.

## Пошук і рекомендації

Система підтримує два взаємодоповнювальні механізми пошуку:

1. **Повнотекстовий пошук**  
   Якщо Elasticsearch увімкнений, пошук виконується за індексованими полями можливостей: назва, опис, eligibility, keywords, disciplines, source. Це дає релевантне ранжування та кращу масштабованість.

2. **Персоналізовані рекомендації**  
   Для авторизованого користувача система формує рекомендації на основі:
   - семантичної близькості профілю та можливості;
   - збігу дисциплін і ключових слів;
   - career stage;
   - країни й mobility-обмежень;
   - дедлайну;
   - попередньої історії користувача: saved, planned, applied, ignored;
   - заповненості профілю.

## Статуси заявки

Кожна можливість може перебувати в одному зі станів:

- **Browsing** — можливість лише переглядається, статус у базі не збережений;
- **Saved** — користувач вважає можливість потенційно цікавою;
- **Planned** — користувач планує подаватися;
- **Applied** — заявку подано;
- **Accepted** — заявку прийнято або фінансування отримано;
- **Rejected** — заявка відхилена;
- **Ignored** — можливість прихована й використовується як негативний сигнал для рекомендацій.

Kanban-дошка дозволяє переміщувати картки між статусами перетягуванням мишею.

## Apply Assistant

Apply Assistant — це допоміжний модуль для підготовки заявки. Він не замінює рішення користувача, а структурує інформацію для аналізу.

Асистент використовує:

- опис конкретної можливості;
- витягнуті вимоги;
- профіль дослідника;
- публікації;
- gap analysis;
- warnings;
- результати вебдослідження;
- AI-мемо.

Результатом є:

- оцінка readiness;
- перелік наступних дій;
- advisor memo;
- draft snippets;
- web research links;
- ризики й аспекти, які треба перевірити вручну.

## Адміністративний модуль

Адміністративна панель призначена для керування джерелами та контролю стану системи. Вона підтримує:

- імпорт Grants.gov;
- імпорт EU Funding & Tenders;
- імпорт зовнішніх RSS/JSON/HTML-джерел;
- перегляд черг задач;
- запуск reminder scan;
- запуск weekly digest;
- запуск high-match alerts;
- оновлення embeddings;
- перегляд health cards для PostgreSQL, Redis, Elasticsearch, worker queue та email provider;
- аналіз дублікатів;
- перегляд адміністративної аналітики.

## Запуск проєкту

### 1. Клонування репозиторію

```powershell
git clone <repository-url>
cd "New project"
```

### 2. Створення Python-середовища

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -e ".[dev,embeddings]"
```

### 3. Встановлення frontend-залежностей

```powershell
cd frontend
npm install
cd ..
```

### 4. Налаштування `.env`

Потрібно створити `.env` на основі прикладу та вказати параметри:

- `DATABASE_URL`;
- `REDIS_URL`;
- `ELASTICSEARCH_URL`;
- `SECRET_KEY`;
- `ORCID_CLIENT_ID`;
- `ORCID_CLIENT_SECRET`;
- `ORCID_REDIRECT_URI`;
- `SMTP_HOST`;
- `SMTP_PORT`;
- `SMTP_USERNAME`;
- `SMTP_PASSWORD`;
- `GROQ_API_KEY`;
- `EMBEDDING_PROVIDER`;
- `FRONTEND_URL`.

### 5. Міграції бази даних

```powershell
.\.venv\Scripts\python.exe -m alembic upgrade head
```

### 6. Запуск повного застосунку

Для локального запуску можна використовувати:

```powershell
.\scripts\run-full-app.ps1
```

Або окремо:

```powershell
.\.venv\Scripts\python.exe -m uvicorn app.main:app --reload
cd frontend
npm run dev
```

## Docker Compose

Для відтворюваного запуску інфраструктури можна використовувати Docker Compose. Він дозволяє підняти PostgreSQL, Redis, Elasticsearch, backend, worker та frontend у спільній мережі.

```powershell
docker compose up --build
```

## Тестування

У проєкті передбачені такі рівні тестування:

- **Unit tests** — перевірка сервісів, рекомендацій, OpenAlex, advisor provider;
- **Integration-style tests** — перевірка API-поведінки, профільного discovery та зовнішніх інтеграцій із моками;
- **Frontend tests** — перевірка React-компонентів, карток можливостей, фільтрів, сторінки деталей.

Запуск backend-тестів:

```powershell
.\.venv\Scripts\python.exe -m pytest
```

Запуск frontend-тестів:

```powershell
cd frontend
npm run test
```

Перевірка production build:

```powershell
cd frontend
npm run build
```

## Висновок

Research Opportunity Matcher демонструє комплексний підхід до побудови сучасної академічної інформаційної системи. Проєкт поєднує backend-архітектуру, frontend UX, інтеграції з відкритими академічними джерелами, повнотекстовий пошук, embeddings, AI-підтримку та workflow-інструменти для підготовки заявок.

З погляду дипломної роботи система має цінність як приклад прикладного програмного комплексу, що вирішує реальну проблему дослідників: пошук релевантного фінансування та організацію процесу подання заявок.
